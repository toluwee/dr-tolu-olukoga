Repeat Search Prediction Model

TensorFlow PySpark MLflow AWS SageMaker Statistical Testing

Overview

Built and deployed deep learning models to predict the likelihood of users performing repeat searches in the travel domain. The model powers recommendation systems serving 1.5M+ monthly users, improving engagement and conversion rates.

Problem Statement

Understanding user search behavior is critical for personalization in travel booking:

  • Engagement Optimization: Need to identify users likely to return and re-search to trigger targeted interventions
  • Recommendation Timing: Optimal moment to present recommendations depends on predicted search likelihood
  • Resource Allocation: Finite email/notification budget requires prioritizing high-propensity users
  • Complex Patterns: Search behavior influenced by dozens of factors (trip characteristics, user history, temporal patterns)

Solution Approach

Developed a multi-stage ML pipeline combining feature engineering, deep learning, and rigorous statistical validation.

Architecture

End-to-end system with three core components:

  • Feature Engineering: 100+ features extracted from search logs, user profiles, and temporal patterns
  • Prediction Model: Deep neural network with attention mechanisms for sequential search behavior
  • Validation Framework: Comprehensive A/B testing and statistical analysis to ensure business impact

Key Features

  • Sequential Modeling: Captures temporal dependencies in user search patterns using recurrent layers
  • Multi-task Learning: Jointly predicts repeat search likelihood and time-to-next-search
  • Feature Importance Tracking: SHAP values for model interpretability and debugging
  • Real-time Inference: Sub-50ms prediction latency for live recommendation serving

Technical Implementation

Technology Stack

  • Data Processing: PySpark for distributed ETL on petabyte-scale search logs
  • Model Development: TensorFlow 2.x with Keras API for deep learning architecture
  • Deployment: AWS SageMaker for model hosting, API Gateway for inference endpoint
  • Experiment Tracking: MLflow for model versioning and metrics logging
  • Validation: Custom statistical testing framework for online metric evaluation

Model Architecture Details

Deep neural network with the following architecture:

  • Embedding layers for categorical features (destination, hotel chain, device type)
  • LSTM layers for sequential search history (last 10 searches)
  • Attention mechanism to identify influential past searches
  • Dense layers with batch normalization and dropout for regularization
  • Multi-task output heads for probability and timing predictions

Feature Engineering

Engineered features across multiple categories:

  • Search Characteristics: Destination popularity, trip duration, lead time, price range
  • User History: Past booking rate, search frequency, device diversity, loyalty status
  • Temporal Patterns: Day of week, seasonality, time since last search, browsing session duration
  • Contextual Signals: Search funnel position, filters applied, sort preferences, page depth

Training & Optimization

  • Dataset: 50M search sessions over 6-month training window
  • Class imbalance handling: Weighted loss function with SMOTE oversampling
  • Hyperparameter tuning: Bayesian optimization with Optuna (500+ trials)
  • Regularization: L2 weight decay, dropout (0.3), early stopping
  • Training time: 8 hours on 4x V100 GPUs

A/B Test Results

  • 8% increase in user re-engagement rate (statistically significant, p < 0.01)
  • 6% improvement in conversion rate for targeted recommendations
  • 14% reduction in irrelevant notifications sent (improved user experience)
  • Positive impact on user satisfaction scores based on post-visit surveys

Lessons Learned

  • Sequential information is critical: Model performance improved significantly when incorporating search history vs. treating searches independently
  • Class imbalance matters: Initial models struggled due to low repeat search rates; proper handling improved precision/recall balance
  • Interpretability builds trust: SHAP explanations helped stakeholders understand and trust model decisions
  • Online metrics > offline metrics: Models with similar offline AUC showed different business impact; always validate with A/B tests
  • Inference latency is non-negotiable: Required significant optimization to meet sub-50ms requirement for real-time serving
