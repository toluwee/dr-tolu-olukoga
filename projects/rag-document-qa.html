RAG-Based Document Q&A System

RAG LangChain NLP GCP Vertex AI Vector Database

Overview

Enterprise-grade NLP question-answering system built for Baptist Health South Florida, processing 500+ policy documents using Retrieval-Augmented Generation (RAG) architecture. The system delivers accurate, cited responses while maintaining 100% citation accuracy and full compliance with healthcare regulatory requirements.

Problem Statement

Healthcare organizations face significant challenges in policy document management:

  • Information Overload: 500+ policy documents across clinical, operational, and compliance domains
  • Time-Consuming Search: Employees spending hours manually searching through documents for specific policies
  • Accuracy Requirements: Healthcare regulations demand 100% accurate citations and source attribution
  • Multi-Document Synthesis: Complex queries often require information from multiple policy documents
  • Compliance Tracking: Need for comprehensive audit trails of information access and retrieval

Solution Approach

Implemented a production RAG system using LangChain and GCP Vertex AI that combines semantic search with large language model reasoning for accurate, contextual responses.

RAG Architecture

Multi-stage pipeline for document processing and retrieval:

  • Document Ingestion: Automated pipeline for processing policy documents (PDF, DOCX, HTML)
  • Intelligent Chunking: Semantic-aware text splitting preserving context boundaries
  • Vector Embedding: Document chunks embedded using state-of-the-art embedding models
  • Retrieval Layer: Hybrid search combining semantic similarity and keyword matching
  • LLM Synthesis: Context-aware response generation with multi-document reasoning
  • Citation System: Automatic source attribution with document references

Key Features

  • Conversation Memory: Multi-turn dialogue with context retention across questions
  • Multi-Document Synthesis: Combines information from multiple sources into coherent responses
  • 100% Citation Accuracy: Every factual claim linked to source document with page numbers
  • Semantic Search: Understanding intent beyond keyword matching
  • Healthcare Compliance: Audit logging and access controls meeting regulatory requirements

Technical Implementation

Technology Stack

  • LLM Platform: GCP Vertex AI for model hosting and inference
  • RAG Framework: LangChain for retrieval and orchestration logic
  • Vector Database: Pinecone/Weaviate for semantic search and retrieval
  • Document Processing: PyMuPDF, python-docx for multi-format parsing
  • Embedding Models: OpenAI embeddings / GCP text-embedding models
  • Backend: Python with FastAPI for REST API endpoints

Document Processing Pipeline

Automated chunking strategy optimized for policy documents:

  • Structure Preservation: Detect and preserve document sections, headers, and hierarchies
  • Semantic Boundaries: Split at natural boundaries (paragraphs, sections) rather than arbitrary character counts
  • Overlap Strategy: 10% chunk overlap to maintain context across boundaries
  • Metadata Enrichment: Tag chunks with document title, section, page number, last updated date
  • Quality Validation: Automated checks for chunk coherence and completeness

Retrieval Strategy

Hybrid approach combining multiple retrieval methods:

  • Dense Retrieval: Vector similarity search using embeddings (primary method)
  • Sparse Retrieval: BM25 keyword matching for specific terms/codes
  • Re-ranking: Cross-encoder model to refine top-k retrieval results
  • Contextual Compression: Filter retrieved chunks for relevance before LLM processing

Conversation Memory Implementation

  • Session-based conversation history tracking
  • Context window management for long conversations
  • Intelligent summarization of older conversation turns
  • Persistence layer for conversation recovery

Citation & Verification System

Ensuring accuracy and traceability:

  • Every response includes source document references with page numbers
  • Inline citations linked to specific retrieved chunks
  • Confidence scoring for generated responses
  • Fallback to "unable to answer" for low-confidence queries
  • Human verification workflow for critical policy questions

Lessons Learned

  • Chunking strategy is critical: Semantic-aware chunking that preserves document structure outperformed fixed-size chunking by 30% in retrieval accuracy
  • Citations build trust: 100% citation accuracy was non-negotiable for healthcare adoption; users need to verify source documents
  • Hybrid retrieval wins: Combining dense and sparse retrieval caught edge cases that either method alone would miss
  • Context windows matter: Conversation memory significantly improved multi-turn dialogue quality
  • Failure modes are important: System must gracefully handle queries it cannot answer rather than hallucinating responses
  • Compliance integration essential: Audit logging and access controls built from day one simplified regulatory approval
